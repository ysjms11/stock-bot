## 1. get_regime — 시장 레짐 판정
===========================================

설명: KR/US 복합점수(0~100) 기반 🟢공격/🟡중립/🔴위기 판정

파라미터:
- mode: "current"(기본) / "history" / "override"
- days: history 모드 시 조회 일수 (기본 5)
- kr_weight: 한국 비중 0~1 (기본 0.6)
- us_weight: 미국 비중 0~1 (기본 0.4)
- regime: override 시 강제 레짐 ("crisis"/"neutral"/"offensive")
- reason: override 사유

반환: regime(🔴/🟡/🟢), combined_score, kr/us 각 신호별 z-score, 디바운스 상태, turbulence

사용 예:
- 일일점검: get_regime → 점수 변화 확인
- 쇼크 강제: get_regime(mode="override", regime="crisis", reason="이란 확전")
- 히스토리: get_regime(mode="history", days=10) → 최근 10일 레짐 추이

## 2. get_alerts — 알림 목록 조회
===========================================

설명: 손절/목표 근접 + 매수감시 + 최근 decision_log

파라미터:

반환:
- alerts: 보유종목 손절/목표 거리 (gap_pct, target_pct)

- recent_decisions: 최근 투자판단 기록

사용 예:
- 일일점검: get_alerts(brief=true) → triggered 있나? 손절 근접?
- 상세확인: get_alerts → 전체 memo 포함

## 3. get_stock_detail — 종목 상세
===========================================

설명: 현재가/PER/PBR/수급/일봉. 한국/미국 자동 판별.

파라미터:
- ticker: 종목코드 ("005930") 또는 미국 ("NVDA")
- tickers: 콤마 구분 다종목 ("005930,000660") 최대 20개
- period: 일봉 조회 ("D60"=60일, "D30", "W20"=20주)
- mode: "volume_profile" / "after_hours" / "orderbook"
- bins: VP 가격 구간 수 (기본 20)
- delay: 다종목 시 딜레이 (기본 0.3초)

반환:
- 기본: price, chg, vol, per, pbr, eps, w52h, w52l, investor(수급 20일)
- volume_profile: POC, VA상단/하단, 가격대별 거래량 분포
- orderbook: 매수/매도 10호가 + 잔량 + 비율
- after_hours: 시간외 현재가/등락률

사용 예:
- 빠른 체크: get_stock_detail(ticker="005930")
- 다종목 일괄: get_stock_detail(tickers="005930,000660,329180")
- 매물대: get_stock_detail(ticker="005930", mode="volume_profile")
- 3년 매물대: get_stock_detail(ticker="005930", mode="volume_profile", period="Y3")
- 일봉 60일: get_stock_detail(ticker="005930", period="D60")
- 호가: get_stock_detail(ticker="005930", mode="orderbook")

## 4. get_supply — 수급 분석
===========================================

설명: 투자자별(외인/기관/개인) 수급 데이터

파라미터:
- mode (필수):
  - "daily": 당일 확정 수급
  - "history": N일 수급 추세 (연속매수/매도 카운트)
  - "estimate": 장중 추정 수급
  - "foreign_rank": 외인 순매수 TOP
  - "combined_rank": 외인+기관 합산 TOP
  - "broker_rank": 증권사별 매매 종목 TOP
- ticker: daily/history/estimate 시 필수
- days: history 일수 (기본 5, 최대 30)
- n: rank 결과 수
- sort: combined_rank/broker_rank 정렬 ("buy"/"sell")
- broker: broker_rank 시 증권사코드

반환:
- daily: 외인/기관/개인 순매수수량/금액
- history: 일별 수급 + 연속 매수/매도 일수
- estimate: 장중 추정치 (장 마감 후에만 유효)
- rank: 순위 목록

사용 예:
- 당일수급: get_supply(mode="daily", ticker="005930")
- 10일추세: get_supply(mode="history", ticker="005930", days=10)
- 스마트머니: get_supply(mode="combined_rank", n=20)
- 증권사별: get_supply(mode="broker_rank", sort="buy", n=10)

주의: mode=history는 장 마감 후에만 데이터 반환. 장중 빈 배열 정상.

## 7. get_macro — 매크로 지표
===========================================

설명: 매크로 대시보드 + 스크리너 통합

파라미터:
- mode:
  - 생략: KOSPI/KOSDAQ/환율 기본
  - "dashboard": VIX/WTI/금/구리/DXY/US10Y/외인수급/이벤트 전체
  - "sector_etf": 섹터 ETF 시세 (8개)
  - "us_sector": 미국 섹터 ETF (SPY/QQQ/XLK 등)
  - "convergence": 이평선 수렴 스크리너 (이격도 포함)
  - "convergence2": 코스닥 위주 하위호환
  - "op_growth": KIS 영업이익 증가율 스크리너
  - "op_turnaround": KIS 적자→흑자 전환
  - "dart_op_growth": DART 기반 연간 OP 성장률
  - "dart_turnaround": DART 기반 적자→흑자
- market: convergence 시 "all"/"kospi"/"kosdaq"
- sort: "spread"/"disp_20"/"disp_60" 또는 "yoy"/"qoq"/"trend"
- spread: 이평 수렴 기준 % (기본 5.0)
- min_growth: 영업이익 최소 증가율 % (기본 50)

반환: 모드별 상이

사용 예:
- 대시보드: get_macro(mode="dashboard")
- 이평수렴: get_macro(mode="convergence", spread=3, sort="disp_20")
- 실적급증: get_macro(mode="op_growth", min_growth=100)
- 턴어라운드: get_macro(mode="op_turnaround")

## 8. get_scan — 전종목 스냅샷 스크리너
===========================================

설명: krx_db 기반 전종목(2,700개) 필터링

파라미터:
- preset: "relative_strength" / "small_cap_buy" / "value" / "momentum" / "oversold" / "foreign_streak"
- market_cap_min/max: 시총 범위 (억원)
- chg_pct_min/max: 등락률 범위 (%)
- per_min/max: PER 범위
- pbr_max: PBR 상한
- foreign_ratio_min: 외인 수급비율 최소
- fi_ratio_min: 외인+기관 합산 최소
- turnover_min: 회전율 최소 (%)
- sort: 정렬 기준 (foreign_ratio/fi_ratio/chg_pct/turnover/market_cap)
- n: 결과 수 (기본 30, 최대 100)
- market: "kospi"/"kosdaq"/"all"
- date: YYYYMMDD (생략 시 최신 DB)

프리셋 상세:
- relative_strength: 시장평균+3% AND fi_ratio>0 (하락장 버틴 종목)
- small_cap_buy: 시총 500~5000억 AND foreign_ratio>0.1%
- value: PER 0~10 AND PBR 0~1 AND 시총>1000억
- momentum: 등락률>3% AND 회전율>1%
- oversold: 등락률 -7% 이하
- foreign_streak: 5일 연속 외인 순매수 + 시총 500억+

사용 예:
- 저평가: get_scan(preset="value")
- 커스텀: get_scan(per_max=15, market_cap_min=5000, fi_ratio_min=0.1, sort="fi_ratio")
- 소형주: get_scan(preset="small_cap_buy", market="kosdaq")

## 9. get_change_scan — 변화 감지 스캔 🆕
===========================================

설명: krx_db N일치 비교로 "변화"를 감지. 천천히 좋아지는 종목 발굴.

파라미터:
- preset: 아래 12개 중 선택
- n: 결과 수 (기본 30)
- market: "kospi"/"kosdaq"/"all"
- 각 프리셋별 임계값 파라미터 (기본값 있음, 변경 가능)

프리셋 목록:
1. ma_convergence — 이평선 수렴 (곧 터짐)
   조건: MA spread <3% + 30일간 수렴 가속
   파라미터: spread_max=3

2. volume_spike — 거래량 폭발
   조건: 10일 평균 거래량 / 20일전 대비 2배+
   파라미터: ratio_min=2.0

3. earnings_disconnect — 실적 좋은데 주가 안 오름
   조건: EPS 90일 +30% 이상인데 YTD 수익률 보합/하락
   파라미터: gap_min=30

4. consensus_undervalued — 컨센서스 대비 저평가
   조건: 목표가 괴리 40%+ AND 커버 3명+
   파라미터: gap_min=40

5. oversold_bounce — 과매도 반등 시작
   조건: RSI < 30 + 전일보다 RSI 상승
   파라미터: rsi_max=30

6. vp_support — 매물대 지지 구간
   조건: VP position < 0.2 (VA 하단 근접)
   파라미터: position_max=0.2

7. golden_cross — 골든크로스 발생
   조건: 오늘 MA5>MA20 + 어제 MA5<MA20

8. sector_leader — 섹터 내 최강
   조건: 섹터 상대강도 > 5%
   파라미터: strength_min=5

9. w52_breakout — 52주 신고가 근접
   조건: 52주 위치 > 0.95
   파라미터: position_min=0.95

10. short_squeeze — 숏커버 진행 중
    조건: 공매도 잔고 10일 -30% 이상 감소
    파라미터: short_change_min=-30

11. credit_unwind — 신용 정리 중 (매도 압력 해소)
    조건: 신용잔고 5일 연속 감소
    파라미터: credit_days=5

12. foreign_reversal — 외인 매수 전환
    조건: 5일 매도 → 매수 전환

13. foreign_accumulation — 외인 보유비율 급증
    조건: 외인 보유비율 5일 +1%p 이상 증가

사용 예:
- 이평수렴: get_change_scan(preset="ma_convergence")
- 임계값 조정: get_change_scan(preset="volume_spike", ratio_min=3.0)
- 복합: "earnings_disconnect 결과 → foreign_reversal 교차"

데이터 요구사항: 과거 데이터 축적 필요.
- 5일 후: MA5, foreign_reversal
- 14일 후: RSI
- 20일 후: 볼린저, golden_cross, 추세 점수
- 60일 후: 매물대(60d), MA60
- 250일 후: 매물대(250d), MA200, 52주 위치

## 22. set_alert — 알림/기록 관리
===========================================

설명: 손절목표/매수감시/투자판단/매매기록/종목비교/삭제

모드별 파라미터:

[손절/목표 등록] (log_type 생략)
- ticker, name, stop_price, target_price, memo

[매수감시 등록] (log_type 생략)
- ticker, name, buy_price, watch_grade(A/B+/B/B-/C+/C/D), memo
- stop_price, target_price도 같이 등록 가능

[투자판단 기록] (log_type="decision")
- date, regime, grades(종목별 등급), actions(액션 목록), notes, watchlist

[매매기록] (log_type="trade")
- ticker, side("buy"/"sell"), qty, price, grade, reason, memo, target_price

[종목비교] (log_type="compare")
- held_ticker, held_score, candidate_ticker, candidate_score, reasoning

[알림삭제] (log_type="delete")
- ticker, market("KR"/"US")

사용 예:
- 손절: set_alert(ticker="267260", name="HD현대일렉트릭", stop_price=800000, target_price=1100000)
- 감시: set_alert(ticker="NVDA", name="NVIDIA", buy_price=171, watch_grade="A", memo="킬질문통과")
- 판단: set_alert(log_type="decision", date="2026-04-08", regime="🔴위기", grades={"005930":{"grade":"B","reason":"실적서프라이즈"}}, actions=["현금보유유지"])
- 매매: set_alert(log_type="trade", ticker="NVDA", side="buy", qty=12, price=171, grade="A", reason="킬질문통과")
- 삭제: set_alert(log_type="delete", ticker="CRSP", market="US")

## 25. get_dart — DART 공시
===========================================

설명: DART 전자공시 조회/사업보고서 저장/읽기

파라미터:
- mode:
  - 생략: 워치리스트 최근 3일 공시
  - "report": 사업보고서 본문 txt 저장 (ticker 지정 가능)
  - "report_list": 저장된 txt 파일 목록
  - "read": 저장된 사업보고서 읽기 (ticker 필수)
  - "insider": 30일 내부자 거래 집계. ticker 필수, days=30 기본. 반환: buy_count, sell_count, net_shares, cluster_flag (3명+ 매수 AND 순매수>0).
- ticker: report/read/insider 시 종목코드
- days: insider 모드 조회 기간 (기본 30)

사용 예:
- 공시확인: get_dart
- 보고서저장: get_dart(mode="report", ticker="005930")
- 보고서읽기: get_dart(mode="read", ticker="005930")
- 7단계 딥서치: get_dart(mode="report") → get_dart(mode="read", ticker="005930")
